# 技能市場使用指南

## 概述

內部技能市場系統讓團隊成員可以分享和安裝專業 AI Agent 技能。所有技能都專注於電信計費系統操作。

## 功能特點

### 📦 技能包管理
- 瀏覽團隊分享的 AI Agent 技能
- 按類別篩選（計費分析、帳單生成、審帳等）
- 搜尋技能名稱、描述、標籤
- 查看技能詳細資訊與評分

### 🔍 智能搜尋
- 即時搜尋過濾
- 按類別快速篩選
- 顯示符合結果數量

### ⭐ 技能資訊
- 作者與組織資訊
- 版本號與更新日期
- 下載次數與評分
- 詳細說明文件（Markdown 格式）
- 技能依賴檢查

### 🌐 市場來源管理
- 支援多個技能來源
- GitHub 倉庫整合
- 自訂市場伺服器
- 啟用/停用來源

## 使用方式

### 1. 開啟技能市場

1. 點擊左側面板的「技能庫」按鈕
2. 切換到「🏪 市場」標籤頁
3. 系統會顯示所有可用的技能包

### 2. 瀏覽技能

**按類別篩選：**
- 左側顯示 6 個專業類別
- 點擊類別查看該類技能
- 類別旁顯示技能數量

**搜尋技能：**
- 使用頂部搜尋框
- 輸入技能名稱、描述或標籤
- 即時顯示符合結果

**類別說明：**
- 📊 計費分析：出帳狀況監控、數據分析
- 📄 帳單生成：發票產生、費用計算
- ✅ 審帳：合規檢查、驗證審核
- 🚨 計費監控：異常偵測、即時預警
- 📈 報表生成：統計圖表、定期報表
- 🗄️ 資料庫營運：DBA 工作、效能優化

### 3. 查看技能詳情

點擊任一技能卡片，右側會顯示：
- 技能完整描述
- 作者與組織資訊
- 版本號與更新時間
- 下載次數與評分
- 授權條款
- 詳細說明文件

### 4. 安裝技能

1. 在右側詳情面板點擊「安裝」按鈕
2. 系統會自動檢查依賴關係
3. 安裝成功後會顯示通知
4. 技能會出現在「我的技能」列表中

### 5. 同步市場

點擊左下角的「🔄 同步」按鈕：
- 從所有啟用的來源抓取最新技能
- 更新技能包資訊
- 顯示同步進度

### 6. 管理來源

點擊「⚙️ 管理來源」按鈕：
- 查看所有配置的市場來源
- 啟用或停用特定來源
- 查看來源類型（GitHub、官方、自訂）

## 市場來源類型

### GitHub 倉庫
- 從 GitHub repository 載入技能
- 自動解析 JSON 格式技能包
- 支援私有倉庫（需要 token）

### 官方市場
- 遠傳電信官方技能市場
- 經過審核的專業技能
- 定期更新維護

### 自訂來源
- 團隊自建的市場伺服器
- API 格式需符合規範
- 可用於內部測試

## 技能包格式

技能市場使用 JSON 格式的技能包：

```json
{
  "version": "1.0",
  "name": "市場名稱",
  "description": "市場說明",
  "packages": [
    {
      "id": "unique-skill-id",
      "skill": {
        // AIAgentSkill 完整定義
        "id": "skill-id",
        "name": "技能名稱",
        "description": "簡短說明",
        // ... 其他技能屬性
      },
      "metadata": {
        "author": "作者姓名",
        "authorEmail": "email@example.com",
        "organization": "組織名稱",
        "version": "1.0.0",
        "downloads": 0,
        "rating": 0,
        "reviews": 0,
        "lastUpdated": 1704528000000,
        "license": "MIT",
        "homepage": "https://...",
        "repository": "https://github.com/...",
        "readme": "# 詳細說明\n\nMarkdown 格式..."
      }
    }
  ]
}
```

## 發布技能到市場

### 方法 1：GitHub Repository

1. 創建 GitHub 倉庫
2. 在根目錄創建 `marketplace.json` 檔案
3. 按照上述格式定義技能包
4. 提交並推送到 GitHub
5. 在系統中添加 GitHub URL 作為來源

**範例 URL：**
```
https://raw.githubusercontent.com/your-org/your-repo/main/marketplace.json
```

### 方法 2：從現有技能匯出

1. 在「我的技能」中選擇要分享的技能
2. 點擊「匯出」按鈕
3. 保存為 JSON 檔案
4. 手動添加 metadata 欄位
5. 上傳到你的市場來源

### 方法 3：自建市場伺服器

創建簡單的 HTTP 伺服器提供 JSON API：

```javascript
// Express.js 範例
app.get('/marketplace.json', (req, res) => {
  res.json({
    version: "1.0",
    packages: [/* 技能包陣列 */]
  });
});
```

## 最佳實踐

### 技能命名
- 使用清晰描述性的名稱
- 避免縮寫和簡稱
- 包含主要功能關鍵字

### 技能描述
- 第一句話說明核心功能
- 列出主要特點（3-5 項）
- 說明使用場景
- 提供範例或截圖

### 標籤選擇
- 使用 3-5 個相關標籤
- 包含技術關鍵字（Oracle、CCBS 等）
- 包含功能關鍵字（分析、監控等）
- 包含領域關鍵字（計費、電信等）

### 版本管理
- 遵循語意化版本（Semantic Versioning）
- 主版本：破壞性變更
- 次版本：新功能
- 修訂版本：錯誤修復

### README 撰寫
使用 Markdown 格式，包含：
- 技能概述
- 功能特點
- 使用方法
- 配置說明
- 常見問題
- 範例截圖

## 故障排除

### 技能無法安裝
- 檢查依賴關係是否滿足
- 確認技能 ID 沒有衝突
- 查看詳細錯誤訊息

### 市場無法同步
- 檢查網路連線
- 確認來源 URL 可訪問
- 驗證 JSON 格式正確

### 技能不顯示
- 確認來源已啟用
- 重新同步市場
- 檢查技能分類是否正確

## 安全性注意事項

### 技能審查
- 安裝前查看技能權限
- 檢查作者資訊可信度
- 查看其他用戶評分與評論

### 權限控制
- 技能執行前需批准危險操作
- 資料庫操作需要確認
- 敏感資訊會被保護

### 來源管理
- 僅添加信任的市場來源
- 定期審查已啟用的來源
- 使用 HTTPS 確保傳輸安全

## 技術支援

遇到問題時：
1. 查看本指南的故障排除章節
2. 檢查技能的 README 文件
3. 聯繫技能作者
4. 向 DevOps 團隊回報
